# Tax Documents — what KC Digital Drive generates

KC Digital Drive is a pass-through donation platform that connects individual donors to verified 501(c)(3) recipient organizations. This doc maps the IRS requirements onto what the codebase actually generates, who's responsible for what, and what's still TODO.

## Audience: donors

### Per-donation receipt (a.k.a. "contemporaneous written acknowledgment")

**Why it exists:** the IRS requires a written acknowledgment from the charity for any donation of **$250 or more** for the donor to claim the deduction (Publication 1771). Best practice is to issue one for every donation regardless of amount.

**Required contents** (all present in `backend/api/services/pdfGenerator.js → generateDonationReceipt`):

| Field | Source |
|-------|--------|
| Donor name | `user_profiles` (or Clerk user, fallback) |
| Donation amount (cash) | `requests.amount` |
| Date of contribution | `requests.fulfilled_at` |
| Recipient organization name | `organizations.name` |
| Recipient EIN | `organizations.ein` |
| 501(c)(3) status statement | hard-coded |
| "No goods or services were provided in exchange" disclaimer | hard-coded |
| Unique receipt number | generated, format `RCPT-YYYY-XXXX` |

**When it fires:** the backend route hooked into the Stripe webhook for a successful payment / request fulfillment should call `generateDonationReceipt(...)`. The webhook handler lives in `backend/api/server.js`; verify the call site if receipts aren't appearing.

**Where it lives:** PDF stored in the `tax-documents` Supabase bucket (path `<donor_id>/<receipt_number>.pdf`). DB row in `donor_documents` with `type='tax_receipt'`.

**Where the donor sees it:** `/donor/documents` → fetched via `fetchDonorDocuments(userId)`. Download via the `/api/documents/download/:documentId` backend route.

### Annual giving statement

**Why it exists:** convenience for donors who itemize. Aggregates all of a year's donations into one document.

**How it's generated:** on demand. Donor clicks a "Generate annual summary" button on `/donor/documents` → frontend calls `generateAnnualSummary(donorId, year)` → backend `POST /api/documents/generate-annual-summary` queries every `tax_receipt` for that donor in that year, totals them, renders one PDF, stores it as `type='annual_summary'`.

**What it contains:**
- Donor info
- Year
- Total amount donated
- List of every donation: date, organization, amount
- Same 501(c)(3) statement

### Donor profile / browse history

Not a tax document — `/donor/dashboard` shows lifetime stats but those aren't a legal record. The receipts and annual summary are.

## Audience: recipient organizations (CBOs)

### Documents the organization uploads (handled in app)

These live in `/cbo/dashboard → Documents` (the `organization_documents` table + bucket):

| Doc | What it proves | Required? |
|-----|----------------|-----------|
| **501(c)(3) Determination Letter** | IRS confirmed the org is tax-exempt | Yes — donors need to know donations are deductible |
| **Tax-exempt certificate** (state-level) | State exemption | Often required for charitable solicitation registration |
| **Annual Report** | Public accountability | Best practice, not required |
| **Financial Statement** | Donor trust | Best practice |
| **Audit Report** | For orgs above threshold | State-specific (CA $2M+, etc.) |
| **Bylaws / Board minutes / Insurance** | Operational | Optional, donor confidence |

### Documents the organization receives elsewhere (NOT generated by KCDD)

- **Form W-9** — Stripe Connect collects this when the org sets up payouts. Stays in Stripe.
- **Form 1099-K** — Stripe issues this annually to the org if payout volume crosses the federal threshold ($600/yr for tax years 2024+; check current year). Stripe handles, KCDD does not need to issue.
- **Form 990 / 990-EZ / 990-N** — annual return the org files directly with the IRS. KCDD doesn't generate.

---

## Audience: the platform itself

- KCDD is not a "merchant of record" — donations are paid org-to-donor through Stripe Connect; the platform does not need to issue 1099-K to donors.
- KCDD should retain receipts + transaction records for at least **3 years** (donor audit window). The PDFs in `tax-documents` plus the `donor_documents` rows + `requests` rows are the audit trail.
- If the platform ever takes a fee, that fee revenue flows to KCDD's own entity and is reported separately on KCDD's own taxes — not via 1099-K to donors or orgs.
